## CRITICAL（App Store リジェクト or 致命的バグ）

### C1: scan.tsx — App-Prefs:PRIVACY でApp Storeリジェクト
- **ファイル**: scan.tsx:276
- **症状**: Apple審査でリジェクト。iOS 18+で無応答の可能性
- **原因**: `Linking.openURL('App-Prefs:PRIVACY&path=CAMERA')` はプライベートAPI
- **修正**: `Linking.openSettings()` に変更
- **ステータス**: [x] 修正済み (9be4dff)

### C2: alarm-optimization.tsx — App-Prefs:SCREEN_TIME でリジェクト
- **ファイル**: alarm-optimization.tsx:156
- **症状**: 同上
- **原因**: `settingsUrl="App-Prefs:SCREEN_TIME"` はプライベートAPI
- **修正**: `app-settings:` に変更
- **ステータス**: [x] 修正済み (9be4dff)

### C3: スヌーズ通知がアラーム解除後も発火する
- **ファイル**: alarmService.ts:164-183, ringing.tsx:128
- **症状**: QRでアラーム解除後、5分後にスヌーズ通知が来る
- **原因**: スヌーズ通知IDがランダム生成で、キャンセル手段がない
- **修正**: 固定ID `${alarm.id}_snooze` を使い、cancelAlarmでキャンセル追加
- **ステータス**: [x] 修正済み (9be4dff)

### C4: ワンタイムアラームが毎日繰り返し発火する
- **ファイル**: alarmService.ts:205-231, scan.tsx
- **症状**: 繰り返しなしのアラームが、解除後も毎日鳴る
- **原因**: 解除フローで alarm.enabled = false にしていない
- **修正**: scan.tsx で解除時に repeatDays.length === 0 なら無効化
- **ステータス**: [x] 修正済み (9be4dff)

## HIGH（機能破損）

### H1: playAlarm 連続呼び出しで音声リーク
- **ファイル**: services内のaudioService相当
- **症状**: 2つのアラーム音が同時再生、片方を止められない
- **原因**: async gap で2回目の呼び出しが1回目のSoundオブジェクトを上書き
- **修正**: mutex ガードを追加
- **ステータス**: [x] 修正済み (9be4dff)

### H2: alarm が null のときスヌーズ画面が壊れる
- **ファイル**: ringing.tsx:123-132
- **症状**: スヌーズ画面で 0:00 表示、通知も来ない
- **原因**: alarm が null のとき scheduleSnooze/saveSnoozeTime をスキップするが画面遷移は実行
- **修正**: alarm null 時にフォールバックスヌーズを設定
- **ステータス**: [x] 修正済み (9be4dff)

### H3: 設定画面のサウンド選択がアラーム編集に漏れる
- **ファイル**: settings.tsx:159, edit.tsx:152-156
- **症状**: 設定でサウンド選択後、別のアラーム編集時にサウンドが勝手に変わる
- **原因**: pendingSound がグローバルストアで、クリアされない
- **修正**: settings.tsx の useFocusEffect で clearPendingSound() を呼ぶ
- **ステータス**: [x] 修正済み (9be4dff)

### H4: デフォルトサウンド設定が完全に機能しない
- **ファイル**: settings.tsx:42, 68-74
- **症状**: デフォルトサウンドの表示が常に "gentle" で変更できない
- **原因**: AsyncStorage への保存/読み込みがない
- **修正**: DEFAULT_SOUND_KEY で永続化
- **ステータス**: [x] 修正済み (9be4dff)

### H5: スキャン画面オープン時にアラーム音が即停止
- **ファイル**: scan.tsx:125-127
- **症状**: QRスキャン前にアラーム音が止まる（スキャン強制の意味がない）
- **原因**: useEffect で mode === 'dismiss' 時に即 stopAlarm()
- **修正**: useEffect の即時 stopAlarm() を削除、スキャン成功時のみ停止
- **ステータス**: [x] 修正済み (9be4dff)

## 教訓（lessons learned）

### なぜ今まで直らなかったか
1. **TypeScript チェックだけでは不十分** — 型は通っても実行時のロジックバグは検出できない
2. **コードレビューがスタイル寄りだった** — 「動くか」ではなく「きれいか」を見ていた
3. **結合テストがない** — 画面遷移やデータフローの問題は単体では見えない
4. **プライベートAPI のリスク理解不足** — App-Prefs: 系は全てリジェクト対象
5. **グローバルステートの管理不備** — pendingSound がスクリーン間で汚染

### 今後の予防策
- コード変更時は「このコードが呼ばれるすべてのパス」をトレースする
- AsyncStorage の読み書きは必ずエラーハンドリングする
- iOS プライベートAPI は `Linking.openSettings()` または `app-settings:` のみ使用
- setTimeout/setInterval は必ず useEffect cleanup で解除する
