## 本日のメインタスク: TestFlight提出

### Step 1: Android Build結果の確認

GitHub Copilotの修正結果を確認する。

```bash
# 最新ビルドの状態確認
eas build:list --platform android --limit 1
```

- **FINISHED** → Step 3へ（Android提出）
- **ERRORED** → エラーログを確認し、Copilotに再指示

### Step 2: iOS Production Build実行

```bash
cd "C:\Users\yuasano\OneDrive - Microsoft\Documents\02 Claude\12-qr-alarm-app"

# OneDrive ACL除去（毎回必須）
powershell -Command "icacls 'C:\Users\yuasano\OneDrive - Microsoft\Documents\02 Claude\12-qr-alarm-app' /remove:d Everyone /T /C"

# iOS Build
eas build --platform ios --profile production --non-interactive
```

**初回ビルド時に聞かれる可能性があること**:
- **Apple ID**: ログインメールアドレスを入力
- **Apple ID Password**: パスワードを入力
- **App-specific password**: 2FAが有効な場合に必要
  - https://appleid.apple.com → サインイン → セキュリティ → App用パスワード → 生成
  - 名前: `EAS Build` として生成
  - 表示されたパスワードをコピーして入力
- **2FAコード**: iPhoneに届く6桁コードを入力
- **Apple Team選択**: 個人アカウントなら自動選択される

ビルド時間: 15-25分。待っている間にStep 4のスクリーンショットを進める。

### Step 3: Android → Google Play Internal Testing 提出

Android Buildが成功していれば:

```bash
eas submit --platform android --profile production
```

**手動の場合**:
1. https://play.google.com/console/ にログイン
2. ScanAlarmアプリを選択（なければ新規作成: パッケージ名 `com.yuasano.qralarm`）
3. **Testing > Internal testing** → **Create new release**
4. AABをアップロード（`eas build:list` で表示されるURLからダウンロード）
5. Release notes: `ScanAlarm初回リリース`
6. **Save** → **Review release** → **Start rollout**
7. **Testers** タブで自分のGoogleアカウントをテスターに追加

### Step 4: iOS → TestFlight 提出

iOS Buildが成功したら:

```bash
eas submit --platform ios --profile production
```

**聞かれる可能性があること**:
- **Apple ID**: メールアドレス
- **App-specific password**: Step 2で生成したものを再利用
- **ASC App ID**: 「新規作成」を選択（初回は自動でApp Store Connectにアプリが作成される）

### Step 5: App Store Connect設定

TestFlightでテスターに配布するための設定:

1. https://appstoreconnect.apple.com にログイン
2. **My Apps** → **ScanAlarm** を選択
3. **TestFlight** タブに移動
4. ビルドが「処理中」→「テスト準備完了」になるのを待つ（数分〜1時間）

**Export Compliance（必須 — これをやらないとテスターに配布できない）**:
5. ビルド番号の横に黄色い警告アイコンが出る → クリック
6. 「このAppは暗号化を使用していますか？」→ **いいえ（No）** を選択
   - ScanAlarmはHTTPS以外の暗号化を使っていないのでNo
7. **保存**

**テスターの追加**:
8. TestFlight > **Internal Testing** の「+」ボタン → 新しいグループ作成
9. グループ名: `ScanAlarm Internal`
10. **テスターを追加** → 自分のApple IDメールアドレスを入力
11. テスト対象ビルドを選択 → **有効にする**

### Step 6: TestFlightでインストール＆テスト

1. iPhoneで **TestFlight** アプリを開く（App Storeから無料ダウンロード）
2. 招待メールが届く → 「View in TestFlight」をタップ
   - またはTestFlightアプリ内に直接表示される
3. **インストール** をタップ
4. ScanAlarmが起動することを確認

**テスト確認項目**:
- [ ] アプリが起動する
- [ ] アラームを新規作成できる
- [ ] アラームが指定時刻に鳴る
- [ ] QRコードスキャンでアラーム停止できる
- [ ] スヌーズが動作する
- [ ] バナー広告が表示される
- [ ] 通知が届く
- [ ] 日本語/英語の切り替えが正しい
